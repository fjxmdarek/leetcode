# 76. 最小覆盖子串 (Minimum Window Substring)

### 题目描述

给你一个字符串 `s` 、一个字符串 `t` 。返回 `s` 中涵盖 `t` 所有字符的最小子串。如果 `s` 中不存在涵盖 `t` 所有字符的子串，则返回空字符串 `""` 。

**注意：**

* 对于 `t` 中重复字符，我们寻找的子字符串中该字符数量必须不少于 `t` 中该字符数量。
* 如果 `s` 中存在这样的子串，我们保证它是唯一的答案。

### 示例

**示例 1：**

> **输入：** s = "ADOBECODEBANC", t = "ABC"
> **输出：** "BANC"
> **解释：** 最小覆盖子串 "BANC" 包含来自字符串 t 的 'A'、'B' 和 'C'。

**示例 2：**

> **输入：** s = "a", t = "a"
> **输出：** "a"

**示例 3：**

> **输入：** s = "a", t = "aa"
> **输出：** ""
> **解释：** t 中两个字符 'a' 均应包含在 s 的子串中，因此没有符合条件的子串，返回空字符串。

### 提示

* `m == s.length`
* `n == t.length`
* `1 <= m, n <= 10^5`
* `s` 和 `t` 由英文字母组成

---

### 解题思路（滑动窗口 + 双哈希表）

这道题的窗口大小是**可变的**，我们需要通过“扩张”寻找可行解，通过“收缩”优化最优解。

1. **初始化**：

   - `need` 哈希表：记录字符串 `t` 中各字符出现的次数。
   - `window` 哈希表：记录当前窗口内各字符出现的次数。
   - `valid` 变量：记录窗口中已经满足 `need` 条件的字符个数。
2. **步骤一：扩张窗口 (Right)**：

   - 不断增加 `right` 指针扩大窗口，直到窗口包含了 `t` 的所有字符（即 `valid == need.size()`）。
3. **步骤二：收缩窗口 (Left)**：

   - 此时窗口已满足条件，开始移动 `left` 指针尝试缩小窗口。
   - 每次移动前，更新最小子串的起始位置和长度。
   - 移出字符时，同步更新 `window` 词频和 `valid` 计数。
4. **循环结束**：

   - 重复上述两个步骤，直到 `right` 到达 `s` 的末尾。

---

### 复杂度分析

* **时间复杂度**：$O(m + n)$。每个字符最多进入窗口一次、离开窗口一次。
* **空间复杂度**：$O(K)$。$K$ 为字符集的大小（本题为 52 个字母），哈希表占用空间是常数级的。
