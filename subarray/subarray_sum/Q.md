# 560. 和为 K 的子数组 (Subarray Sum Equals K)

### 题目描述

给你一个整数数组 `nums` 和一个整数 `k` ，请你统计并返回该数组中和为 `k` 的子数组的个数。

子数组是数组中元素的 **连续** 非空序列。

### 示例

**示例 1：**

> **输入：** nums = [1,1,1], k = 2
> **输出：** 2
> **解释：** [1,1] 和 [1,1] 都是和为 2 的子数组。

**示例 2：**

> **输入：** nums = [1,2,3], k = 3
> **输出：** 2
> **解释：** [1,2] 和 [3] 都是和为 3 的子数组。

### 提示

* `1 <= nums.length <= 2 * 10^4`
* `-1000 <= nums[i] <= 1000`
* `-10^7 <= k <= 10^7`

---

### 解题思路（前缀和 + 哈希表）

这道题虽然看似可以用滑动窗口，但由于数组中存在 **负数**，窗口的左右移动不再具有单调性（即窗口变长和不一定变大），因此最稳妥的解法是 **前缀和**。

1. **核心逻辑**：

   - 设 $pre[i]$ 为从索引 $0$ 到 $i$ 的元素之和。
   - 任意子数组 $[j..i]$ 的和可以表示为 $pre[i] - pre[j-1]$。
   - 我们要找的是满足 $pre[i] - pre[j-1] = k$ 的子数组，等价于查找 $pre[j-1] = pre[i] - k$。
2. **哈希表加速**：

   - 在遍历数组计算当前前缀和 `curr_sum` 时，我们利用哈希表记录之前出现过的所有前缀和及其出现的次数。
   - 每次计算出 `curr_sum`，只需在哈希表中查找是否存在 `curr_sum - k`。
   - 如果存在，则说明找到了对应数量的子数组，将其出现的次数累加进结果。
3. **初始化**：

   - 哈希表需要预存 `{0: 1}`。这代表前缀和为 0 出现过 1 次（处理子数组恰好从索引 0 开始且和为 k 的情况）。

---

### 时间复杂度

$O(n)$，只需遍历一次数组。空间复杂度为 $O(n)$，用于存储哈希表。
