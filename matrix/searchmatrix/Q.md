# 240. 搜索二维矩阵 II (Search a 2D Matrix II)

### 题目描述

编写一个高效的算法来搜索 `m x n` 矩阵 `matrix` 中的一个目标值 `target` 。该矩阵具有以下特性：

* 每行的元素从左到右升序排列。
* 每列的元素从上到下升序排列。

### 示例

**示例 1：**

> **输入：** matrix = [[1,4,7,11,15],[2,5,8,12,19],[3,6,9,16,22],[10,13,14,17,24],[18,21,23,26,30]], target = 5
> **输出：** true

**示例 2：**

> **输入：** matrix = [[1,4,7,11,15],[2,5,8,12,19],[3,6,9,16,22],[10,13,14,17,24],[18,21,23,26,30]], target = 20
> **输出：** false

### 提示

* `m == matrix.length`
* `n == matrix[i].length`
* `1 <= n, m <= 300`
* `-10^9 <= matrix[i][j] <= 10^9`
* `-10^9 <= target <= 10^9`

---

### 解题思路（Z 字形查找 / 站在“路口”思考）

如果从左上角 $(0,0)$ 开始找，向右和向下都是变大，我们无法决定走哪条路。
关键在于：找到一个**“单调性相反”**的起点。

#### 核心逻辑：从右上角 $(0, n-1)$ 开始

1. **初始化**：指针 `i = 0`, `j = n - 1`。
2. **比较与移动**：
   - 如果 `matrix[i][j] == target`：找到目标，返回 `true`。
   - 如果 `matrix[i][j] > target`：说明这一列下面的数都比目标大（因为列是升序的），所以**剔除这一列**，`j--`。
   - 如果 `matrix[i][j] < target`：说明这一行左边的数都比目标小（因为行是升序的），所以**剔除这一行**，`i++`。
3. **结束条件**：当 `i` 或 `j` 越界时，说明矩阵中不存在该目标，返回 `false`。

---

### 复杂度分析

* **时间复杂度**：$O(m + n)$。在最坏的情况下，我们会从右上角走到左下角，总共移动 $m+n$ 次。
* **空间复杂度**：$O(1)$。只使用了两个指针变量。
