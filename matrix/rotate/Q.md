# 48. 旋转图像 (Rotate Image)

### 题目描述

给定一个 `n x n` 的二维矩阵 `matrix` 表示一个图像。请你将图像顺时针旋转 90 度。

你必须在 **原地** 旋转图像，这意味着你需要直接修改输入的二维矩阵。**请不要** 使用另一个矩阵来旋转图像。

### 示例

**示例 1：**

> **输入：** matrix = [[1,2,3],[4,5,6],[7,8,9]]
> **输出：** [[7,4,1],[8,5,2],[9,6,3]]

**示例 2：**

> **输入：** matrix = [[5,1,9,11],[2,4,8,10],[13,3,6,7],[15,14,12,16]]
> **输出：** [[15,13,2,5],[14,3,4,1],[12,6,8,9],[16,7,10,11]]

### 提示

* `n == matrix.length == matrix[i].length`
* `1 <= n <= 20`
* `-1000 <= matrix[i][j] <= 1000`

---

### 解题思路

直接进行位置变换 `(i, j) -> (j, n-1-i)` 的原地交换逻辑非常复杂。一个更优雅、更不容易出错的办法是利用 **矩阵对称性** 进行组合变换。

#### 方法：主对角线翻转 + 左右翻转

顺时针旋转 90 度等价于：

1. **主对角线翻转**：将 `matrix[i][j]` 与 `matrix[j][i]` 交换。
   - 此时矩阵变成了它的**转置矩阵**。
   - 例如：
     ```
     1 2 3      1 4 7
     4 5 6  ->  2 5 8
     7 8 9      3 6 9
     ```
2. **每一行左右翻转**：将 `matrix[i][j]` 与 `matrix[i][n-1-j]` 交换。
   - 翻转后：
     ```
     1 4 7      7 4 1
     2 5 8  ->  8 5 2
     3 6 9      9 6 3
     ```
   - 结果即为顺时针旋转 90 度。

---

### 复杂度分析

* **时间复杂度**：$O(N^2)$。对角线翻转需要遍历一半矩阵，左右翻转需要遍历每一行的一半。
* **空间复杂度**：$O(1)$。所有操作都在原矩阵上通过交换完成。
