# 141. 环形链表 (Linked List Cycle)

### 题目描述

给你一个链表的头节点 `head` ，判断链表中是否有环。

如果链表中有某个节点，可以通过连续跟踪 `next` 指针再次到达，则链表中存在环。 为了表示给定链表中的环，评测系统内部使用整数 `pos` 来表示链表尾连接到链表中的位置（索引从 0 开始）。**注意：`pos` 不作为参数进行传递** 。仅仅是为了标识链表的实际情况。

如果链表中存在环 ，则返回 `true` 。 否则，返回 `false` 。

### 示例

**示例 1：**

> **输入：** head = [3,2,0,-4], pos = 1
> **输出：** true
> **解释：** 链表中有一个环，其尾部连接到第二个节点。

**示例 2：**

> **输入：** head = [1,2], pos = 0
> **输出：** true
> **解释：** 链表中有一个环，其尾部连接到第一个节点。

**示例 3：**

> **输入：** head = [1], pos = -1
> **输出：** false
> **解释：** 链表中没有环。

### 提示

* 链表中节点的数目范围是 `[0, 10^4]`
* `-10^5 <= Node.val <= 10^5`
* `pos` 为 `-1` 或者链表中的一个 **有效索引** 。

### 进阶

* 你能用 $O(1)$（即，常量）内存解决此问题吗？

---

### 解题思路（快慢指针 / Floyd 判圈算法）

想象两个人在圆形跑道上赛跑：只要跑道有环，速度快的人一定会在某个时刻“套圈”速度慢的人。

1. **初始化**：

   - 设置两个指针，`slow`（慢指针）和 `fast`（快指针），初始都指向 `head`。
2. **移动规则**：

   - `slow` 每次走 **1** 步：`slow = slow.next`。
   - `fast` 每次走 **2** 步：`fast = fast.next.next`。
3. **判断逻辑**：

   - 如果链表无环：`fast` 最终会遇到 `null`，此时直接返回 `false`。
   - 如果链表有环：由于 `fast` 比 `slow` 跑得快，且每次缩短 1 步的距离，两者最终一定会在环内的某个节点 **相遇**（即 `fast == slow`）。

---

### 复杂度分析

* **时间复杂度**：$O(n)$。
  - 无环时，`fast` 走 $n/2$ 步到末尾。
  - 有环时，`fast` 进入环后，与 `slow` 的距离每走一步缩小 1，最多在环的一圈内相遇。
* **空间复杂度**：$O(1)$。仅使用了两个指针变量，不需要额外存储节点信息。
