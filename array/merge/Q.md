# 56. 合并区间 (Merge Intervals)

### 题目描述

以数组 `intervals` 表示若干个区间的集合，其中单个区间为 `intervals[i] = [starti, endi]` 。请你合并所有重叠的区间，并返回 一个不重叠的区间数组，该数组需恰好覆盖输入中的所有区间 。

### 示例

**示例 1：**

> **输入：** intervals = [[1,3],[2,6],[8,10],[15,18]]
> **输出：** [[1,6],[8,10],[15,18]]
> **解释：** 区间 [1,3] 和 [2,6] 重叠, 将它们合并为 [1,6].

**示例 2：**

> **输入：** intervals = [[1,4],[4,5]]
> **输出：** [[1,5]]
> **解释：** 区间 [1,4] 和 [4,5] 可被视为重叠区间。

### 提示

* `1 <= intervals.length <= 10^4`
* `intervals[i].length == 2`
* `0 <= starti <= endi <= 10^4`

---

### 解题思路（排序 + 一次遍历）

如果不排序，我们很难判断哪些区间可以合并。**排序**是解决区间问题的“金钥匙”。

1. **排序**：

   - 按照区间的 **左端点** (`start`) 进行升序排序。
   - 排序后，能够合并的区间在数组中一定是 **连续** 的。
2. **合并逻辑**：

   - 初始化一个空列表 `merged` 用于存储结果。
   - 遍历排序后的区间：
     - 如果 `merged` 为空，或者当前区间的左端点 **大于** `merged` 最后一个区间的右端点，说明不重叠，直接将当前区间加入 `merged`。
     - 否则（当前区间左端点 $\le$ `merged` 最后一个区间的右端点），说明有重叠，我们需要 **更新** `merged` 最后一个区间的右端点，取两者的最大值：`max(merged[-1][1], interval[1])`。

---

### 复杂度分析

* **时间复杂度**：$O(n \log n)$。主要开销是排序。随后的线性遍历为 $O(n)$。
* **空间复杂度**：$O(\log n)$ 或 $O(n)$。取决于排序算法使用的栈空间。
